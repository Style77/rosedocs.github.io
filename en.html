Arguments

Every argument has to starts with < and ends with >

  • @USER - mentions new user
  • USER - nick of new user
  • GUILD - server name
  • GUILD_COUNT - server members count
  • GUILD_COUNT+ - server members count + 1 new
  • GUILD_OWNER - server owner

Commands

{argument} <- this means that the argument is optional.

[argment] <- this means that the argument is required.

Tags

  • tags (shows you the tags available in a guild)

  • tag

    • create/add [name] {- content} (- content is optional, if you don't add it in, you can add it after bot instructs you to do so)

      Example: tag add xxxxxxxxxxxxxx

      Example 2: tag create xxxxxxxx - the content goes here in the tag

    • edit [name] (instructions will be sent by the bot. Don't think I need to explain this one)

    • remove [name] (that's right, it removes a tag, permanently)

    • info [name] (does exactly what it says on the package)

    • claim [name] (when the owner of the tag isn't in the guild anymore, you can claim ownership of a tag via this command)

    • global [guild_id] [name]

      • info [guild_id] [name]
